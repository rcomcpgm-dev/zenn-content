VS CodeのターミナルからClaude Codeを起動すれば、コード編集とAI指示を同じ画面で行える最強の開発環境が手に入ります。セットアップから実践的なワークフローまで解説します。

## なぜVS Code × Claude Codeなのか

Claude CodeはCLIツールなので、どのエディタとも組み合わせられます。その中でも **VS Code** との相性が抜群です：

- VS Code内蔵のターミナルからClaude Codeを起動できる
- ファイルの変更がリアルタイムでエディタに反映される
- 拡張機能と組み合わせて最強の環境になる

## セットアップ

### 1. VS Codeのターミナルを開く

VS Codeで `Ctrl + `` （バッククォート）を押すとターミナルが開きます。

### 2. プロジェクトフォルダで起動

```bash
claude
```

これだけです。VS Codeで開いているプロジェクトのフォルダでClaude Codeが起動します。

### 3. 画面レイアウト

おすすめのレイアウト：

```
┌─────────────────────────────────┐
│  エディタ（コード表示）           │
│                                 │
├─────────────────────────────────┤
│  ターミナル（Claude Code）       │
│                                 │
└─────────────────────────────────┘
```

上半分でコードを確認しながら、下半分でClaude Codeに指示を出す。AIが変更したファイルは上のエディタにリアルタイムで反映されます。

## 便利なワークフロー

### エラーをそのまま貼る

VS Codeのエディタに赤い波線（エラー）が出たら：

```
TypeScriptのエラーが出てる。直して。
```

Claude Codeがプロジェクト内のエラーを検出して修正してくれます。

### ターミナルを分割する

`Ctrl + Shift + 5` でターミナルを分割できます：

- **左ターミナル**: Claude Code
- **右ターミナル**: `npm run dev`（開発サーバー）

AIに指示しながら、リアルタイムで動作確認ができます。

### VS Code拡張機能との組み合わせ

| 拡張機能 | Claude Codeとの相乗効果 |
|----------|------------------------|
| ESLint | AIが書いたコードのlintエラーを即座に発見 |
| Prettier | 自動フォーマットでコードを整形 |
| GitLens | AIが変更した箇所のgit履歴を確認 |
| Error Lens | エラーをインラインで表示 → AIに修正を依頼しやすい |

## Claude Code拡張機能

Anthropic公式の **Claude Code VS Code拡張機能** もあります：

1. VS Codeの拡張機能マーケットプレイスで「Claude Code」を検索
2. インストール
3. サイドバーにClaude Codeのアイコンが追加される

ターミナルでの操作が苦手な方は、この拡張機能から使うのもありです。

## 実践的な開発フロー

### 1. 新機能の開発

```
1. VS Codeでプロジェクトを開く
2. ターミナルでClaude Codeを起動
3. 「ユーザー登録画面を作って」と指示
4. AIがファイルを作成 → エディタに即反映
5. 開発サーバーでブラウザ確認
6. 修正点をClaude Codeに伝える
7. 完成したらコミット
```

### 2. バグ修正

```
1. エラーメッセージをコピー
2. Claude Codeに貼り付け
3. 「このエラーを直して」
4. 修正を確認 → コミット
```

### 3. コードレビュー

```
1. 「このファイルをレビューして」
2. 改善点を提案してくれる
3. 「その改善を適用して」
4. 完了
```

## まとめ

- VS Codeのターミナルから `claude` で起動するだけ
- エディタとターミナルを同時に見られるのが最大の利点
- ターミナル分割でAI + 開発サーバーを並列運用
- 公式VS Code拡張機能もある
