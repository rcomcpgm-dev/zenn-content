Claude Codeを毎日使う中で見つけた、生産性を劇的に上げるテクニックを10個厳選しました。初心者から中級者まで即実践できます。

## はじめに

Claude Codeを毎日使っている中で見つけた、**生産性を爆上げするテクニック**を10個紹介します。初心者から中級者まで使えるものを厳選しました。

## 1. 指示は具体的に書く

```
# ❌ ダメな指示
いい感じのフォームを作って

# ✅ 良い指示
メールアドレスとパスワードのログインフォームを作って。
バリデーションはメール形式チェックとパスワード8文字以上。
Tailwind CSSでスタイリング。送信ボタンはloading状態も実装して。
```

**具体的であればあるほど、AIの出力精度が上がります。**

## 2. エラーはそのまま貼る

エラーが出たら、メッセージをそのままClaude Codeに貼り付けてください：

```
このエラーが出た：
TypeError: Cannot read properties of undefined (reading 'map')
```

自分で原因を推測する必要はありません。AIがスタックトレースとコードを照合して原因を特定してくれます。

## 3. /compactを活用する

長い作業をしていると会話が長くなり、AIの応答が遅くなったり精度が落ちたりします。

```
/compact
```

このコマンドで会話を要約・圧縮できます。**作業が一段落したタイミングで打つ**のがおすすめです。

## 4. 段階的に作る

一度に大きなものを頼むより、段階的に作る方が精度が高いです：

```
# ❌ 一気に頼む
ECサイトを作って

# ✅ 段階的に
1回目: 商品一覧ページを作って
2回目: 商品詳細ページを追加して
3回目: カート機能を実装して
4回目: 決済画面を追加して
```

## 5. 「〜を参考にして」が強い

既存のコードを参考にさせると、プロジェクトの規約に沿ったコードを書いてくれます：

```
UserList.tsxを参考にして、ProductList.tsxを作って
```

ゼロから作るより、既存コードをベースにした方がプロジェクト内の一貫性が保たれます。

## 6. テストを先に書かせる

テスト駆動開発（TDD）をAIでやると効率的です：

```
1. まずUserService のテストを書いて
2. テストが通るようにUserServiceを実装して
```

テストを先に書くことで、AIが「何を作ればいいか」を明確に理解します。

## 7. git diffで変更を確認する習慣

AIがコードを変更した後は、必ず差分を確認しましょう：

```
git diff を見せて
```

想定外の変更が入っていないかチェックしてから、コミットする癖をつけてください。

## 8. CLAUDE.mdに失敗パターンを記録

AIが同じミスを繰り返すなら、CLAUDE.mdに書いておきます：

```markdown
## 注意事項
- useEffectの依存配列を空にしないこと
- APIのURLは環境変数から取得すること（ハードコーディング禁止）
- importは相対パスではなく@/エイリアスを使うこと
```

一度書けば、以降は同じミスをしなくなります。

## 9. 複数ファイルの一括変更

Claude Codeの真価は複数ファイルの同時変更です：

```
全コンポーネントのimport文を、相対パスから@/エイリアスに変更して
```

手作業なら1時間かかる修正を、数秒で終わらせてくれます。

## 10. 「説明して」を使いこなす

コードを書かせるだけでなく、理解のために使うのも有効です：

```
このプロジェクトのアーキテクチャを説明して
src/lib/auth.ts が何をしているか説明して
このエラーが起きる原因を説明して
```

チームに新しく入ったメンバーに説明する代わりに、Claude Codeに説明してもらうこともできます。

## ボーナス：やってはいけないこと

- **秘密情報を含むファイルを読ませない** — `.env` のAPIキーなど
- **本番環境で直接使わない** — 必ず開発環境で確認してから
- **AIの出力を無条件で信じない** — 必ず動作確認する
- **著作権のあるコードをそのまま使わせない** — ライセンスに注意

## まとめ

1. 指示は具体的に
2. エラーはそのまま貼る
3. /compactで会話を整理
4. 段階的に作る
5. 既存コードを参考にさせる
6. テストを先に書かせる
7. git diffで確認する
8. CLAUDE.mdに失敗パターンを記録
9. 複数ファイルの一括変更に活用
10. 「説明して」で学習にも使う
