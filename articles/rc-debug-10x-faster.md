Claude Codeにエラーメッセージを貼るだけで、原因特定から修正まで自動で行ってくれます。従来のGoogle検索→試行錯誤のループを一気にショートカットする方法を解説します。

## バグ修正が10倍速くなる理由

従来のバグ修正：
1. エラーメッセージを読む
2. Google検索 / Stack Overflowで調べる
3. 原因を推測する
4. 修正してテスト
5. ダメなら2に戻る

Claude Codeでのバグ修正：
1. エラーメッセージを貼る
2. **AIが原因特定 → 修正まで完了**

## パターン1：エラーメッセージをそのまま貼る

```
このエラーが出た：

TypeError: Cannot read properties of undefined (reading 'map')
  at UserList (src/components/UserList.tsx:15:20)
```

Claude Codeの対応：
1. 該当ファイルを読む
2. エラー箇所を特定
3. 原因を説明（例：APIレスポンスが未到達の状態でmapを呼んでいる）
4. 修正コードを書く（例：オプショナルチェーン `users?.map()` を追加）

**自分で調べる時間ゼロ**です。

## パターン2：「動かない」を伝える

エラーメッセージがなくても大丈夫：

```
ログインボタンを押しても何も起きない。
コンソールにエラーも出てない。
```

Claude Codeの対応：
1. ログイン関連のコードを読む
2. イベントハンドラの接続を確認
3. 原因を特定（例：`onClick` がバインドされていない、`async` が抜けている等）
4. 修正

## パターン3：スクリーンショットを見せる

```
この画面でレイアウトが崩れてる。直して。
（スクリーンショットを添付）
```

Claude Codeが画像を解析して、CSSの問題を特定・修正してくれます。

## パターン4：テストが通らない

```
npm run test を実行して。失敗しているテストがあったら直して。
```

テストの実行 → 失敗箇所の特定 → コードの修正 → 再テストまで自動で回してくれます。

## パターン5：ビルドエラー

```
npm run build でエラーが出る。全部直して。
```

TypeScriptの型エラー、importの不備、設定ファイルの問題など、ビルドが通るまで修正を繰り返してくれます。

## 効果的なバグ報告の書き方

AIに伝えるときも「良いバグ報告」の方が精度が上がります：

```
# ❌ 悪い例
動かない

# ✅ 良い例
「投稿を保存」ボタンを押すと画面が白くなる。
コンソールに "Uncaught TypeError: title.trim is not a function" と出ている。
titleが空の状態で保存しようとすると再現する。
```

ポイント：
- **何をしたか**（操作手順）
- **何が起きたか**（症状）
- **エラーメッセージ**（あれば）
- **再現条件**（わかれば）

## 複雑なバグの対応

1つの指示で解決しない場合は、段階的に進めます：

```
1回目: 「このエラーの原因を調査して。まだ修正しないで。」
2回目: 「原因はわかった。修正案を3つ提案して。」
3回目: 「案2で修正して。」
```

Plan Modeを使うのも効果的です。

## まとめ

- エラーメッセージは**そのまま貼る**だけでOK
- エラーがなくても**症状を伝える**だけで原因特定
- テスト実行→修正→再テストも自動
- 良いバグ報告を書くとAIの精度も上がる
