プログラミング未経験からClaude Codeだけを使い、約16時間で実際に動くWebサービスを作った体験談です。成功と失敗のリアルな記録を共有します。

## はじめに

プログラミング未経験の状態からClaude Codeだけを使って、約16時間で実際に動くWebサービスを作りました。何がうまくいき、何につまずいたかを正直に共有します。

## 作ったもの

**シンプルな予約管理システム**
- お客さんがWeb上で日時を選んで予約できる
- 管理者が予約一覧を確認・管理できる
- メール通知あり

美容院をやっている知人に「予約管理が大変」と聞いて、作ることにしました。

## 私のスペック（開発前）

- HTML/CSSの存在は知っている程度
- JavaScriptは書いたことがない
- ターミナルは触ったことがなかった
- ExcelのVLOOKUPが限界

## 開発の過程

### Day 1：環境構築

このシリーズの記事を参考に、Node.jsとClaude Codeをインストール。所要時間30分。

### Day 2-3：フロントエンド

```
Next.jsで予約システムのフロントエンドを作って。

画面：
- トップページ（サービス紹介）
- 予約画面（日時選択カレンダー + 名前・電話番号入力）
- 予約完了画面
- 管理者用ログイン画面
- 管理者用予約一覧画面
```

Claude Codeが全画面を生成。途中で「カレンダーの見た目を変えて」「スマホ対応して」と微調整を繰り返しました。

### Day 4-5：バックエンド

```
Supabaseをバックエンドとして設定して。

テーブル：
- reservations（予約情報）
- services（サービスメニュー）
- blocked_dates（休業日）

予約のCRUD APIを作って。
```

「Supabaseって何？」の状態でしたが、Claude Codeが設定から接続まで全部やってくれました。

### Day 6-7：仕上げとデプロイ

```
予約が入ったら管理者にメール通知を送る機能を追加して。
Resendを使って。
```

```
Vercelにデプロイして。
```

## うまくいったこと

### 1. 0からの環境構築がスムーズ
Claude Codeのおかげで「何をインストールすればいいかわからない」問題がゼロ。

### 2. エラーを自分で解決しなくていい
エラーが出るたびにClaude Codeに貼り付けるだけ。Google検索で何時間も彷徨うことがなかった。

### 3. デザインの調整が楽
「もうちょっと青を濃くして」「ボタンを丸くして」で即反映。

## つまずいたこと

### 1. 指示が曖昧だと想定と違うものができる
「いい感じの予約画面」→ 想定と全然違うものが出てきた。**具体的に指示する**のが重要。

### 2. 大きな変更は壊れやすい
一度にたくさんの機能を頼むと、既存の部分が壊れることがあった。**小さく進める**のがコツ。

### 3. コードの中身が理解できない
動いてはいるけど、何が書かれているか理解できない。メンテナンスに不安が残る。

## 学んだこと

1. **AIがあってもプログラミングの基礎知識は役に立つ**
2. **小さく作って動かして改善するのが最も効率的**
3. **完璧を目指さない。動けばいい（最初は）**
4. **バージョン管理（Git）は絶対に最初からやるべき**

## 未経験から作りたい人へのアドバイス

1. まずこのシリーズの入門編を読んで環境構築
2. 最初は小さいものから（Todoアプリ、メモアプリ）
3. 慣れたら自分が欲しいものを作る
4. 完璧じゃなくても公開する。フィードバックが最大の学び
