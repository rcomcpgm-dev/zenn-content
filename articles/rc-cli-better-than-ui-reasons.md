「なぜわざわざ黒い画面で使うの？」と思うかもしれません。結論から言うと、コーディング作業においてCLIはUIより圧倒的に生産性が高いです。その5つの理由を解説します。

## 理由1：ファイル操作が直接できる

### UIの場合
1. AIにコードを生成してもらう
2. 出力されたコードをコピー
3. エディタに切り替えてペースト
4. ファイルを保存

### CLIの場合
1. AIに指示を出す
2. **AIが直接ファイルを作成・編集する**

コピペ作業がゼロ。Claude Codeはあなたのプロジェクトフォルダに直接アクセスして、ファイルの作成・編集・削除をすべて自動で行います。

## 理由2：プロジェクト全体を理解できる

ChatGPTやClaude（Web版）は、あなたが貼り付けたコードしか見えません。

Claude Codeは **プロジェクトのフォルダ構成、設定ファイル、依存関係をすべて読める** ので、文脈を理解したうえで適切なコードを書いてくれます。

```
# UIだとこう聞かないといけない
「Reactで、既存のApp.tsxにHeaderコンポーネントを追加して。
ちなみにApp.tsxの中身はこれで、routerの設定はこうで、
CSSはTailwindを使っていて...」

# CLIなら
「Headerコンポーネントを追加して」
```

これだけで、既存のコードを読んで適切な場所に適切なコードを書いてくれます。

## 理由3：コマンド実行ができる

Claude Codeはコードを書くだけでなく、**ターミナルコマンドも実行できます**：

- `npm install` でライブラリのインストール
- `npm run build` でビルドの実行
- `git commit` でコミット
- テストの実行

UIチャットでは「このコマンドを実行してください」と言われて自分でコピペする必要がありますが、Claude Codeは**自分で実行**します。

## 理由4：会話のコンテキストが途切れない

Web UIでは、長い会話になるとコンテキストウィンドウの制限で過去の内容を忘れます。

Claude Codeには `/compact` コマンドがあり、会話を要約して圧縮できます。これにより、長時間の作業でもコンテキストを維持できます。

さらに **CLAUDE.md** というファイルにプロジェクトの情報を書いておけば、毎回の会話で自動的に読み込まれます（これは別記事で詳しく解説します）。

## 理由5：自動化・連携がしやすい

CLIは他のツールとの連携が簡単です：

- **Git** との連携でバージョン管理
- **GitHub Actions** でCI/CDパイプライン
- **VS Code** のターミナルから直接利用
- **MCPサーバー** で外部サービスと接続

UIではボタンをポチポチするしかありませんが、CLIならスクリプトで自動化できます。

## UIが向いているケース

もちろんUIにも良い点はあります：

- **ちょっとした質問** → ChatGPTやClaude Webの方が手軽
- **コードを書かない用途** → 文章作成、翻訳、要約など
- **スマホから使いたい** → CLIはPC必須

**コーディング作業ならCLI一択** というのが結論です。

## 比較まとめ

| 観点 | UI（Web版） | CLI（Claude Code） |
|------|-------------|-------------------|
| ファイル操作 | コピペ必要 | 直接操作 |
| プロジェクト理解 | 貼った分だけ | 全体を把握 |
| コマンド実行 | 手動 | 自動 |
| コンテキスト維持 | 制限あり | compact + CLAUDE.md |
| 自動化 | 不可 | Git, CI/CD連携可 |
| 手軽さ | ブラウザだけ | Node.js環境が必要 |

## まとめ

CLIは「不便な古い方法」ではなく、**AIコーディングにおいては最も効率的なインターフェース**です。

最初のハードルを越えれば、UIには戻れなくなるはずです。
